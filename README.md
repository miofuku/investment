# 价值投资研究工作区(investment)

按**市场**分模块的价值投资研究系统集合。每个市场一个独立子目录,共享同一套方法论与工程纪律:
**量化做广度筛选 + LLM 做价值深度尽调 + 纯静态网页发布,全流程数据本地生成。**

> **共同定位:这是纪律严明的"研究助手",不是已验证的"交易策略"。**
> 它帮你高效缩小范围、系统排雷、逼你看清假设;每个买卖决定仍需你自己的判断兜底。

## 共享设计哲学(每个市场都遵守)

1. **数字全在确定性代码里算**,LLM 只做编排与定性表达,绝不做算术或编造数字。
2. **工具诚实标注边界**:算不准就返回 null + 警示,不硬给"看着可信"的错数。
3. **客观与解读分离**:资金面/宏观只报可查证的客观数据,不编码"意图"叙事。
4. **不下买卖结论、不给目标价**:摊开假设与风险,决策留给人。
5. **本地生成 + 静态发布**:数据/简报在本地跑(数据源限速、密钥在本地),只把静态产物部署到 CDN。
6. **不做回测**:免费数据无 point-in-time、无退市股(前视/幸存者偏差),回测不可信——刻意不做。

## 目录约定

每个市场目录**自包含**一条完整流水线,典型形态(以 china-a 为参照):

```
<market>/
  ├─ 数据/筛选脚本        # 洗池 → 因子打分 → 候选缩小,产 CSV
  ├─ agent_*             # 链式 import 的 LLM 深挖工具集,产结构化简报
  ├─ push_to_sheets.py   # 本地产出 → Google Sheets + 生成 data.js
  ├─ index.html / data.js# 纯静态前端 + 数据包(data.js 为生成产物)
  ├─ build_and_deploy.sh # 一键:生成数据 → 部署到 Cloudflare Pages
  ├─ .env                # 密钥/配置(gitignore),如 ZAI_API_KEY、REQUEST_ENDPOINT
  ├─ service_account.json# gspread 凭证(gitignore)
  └─ README.md           # 该市场的详细说明
```

**约定:**
- **密钥与配置放各目录自己的 `.env`**(gitignore),脚本启动自动加载,不写死在代码/前端里。
- **生成产物不进 git**:`data.js`、`*.csv` 缓存、`*.json` 凭证、`.env` 均已忽略(见各目录 `.gitignore`)。
- **部署用 Wrangler 直传**本地的 `index.html` + `data.js`,GitHub 不参与部署链路。
- 跨市场暂不强行抽象公共库;先把第二个市场跑通,再把真正重复的部分上提为共享模块(勿过度工程)。

---

## 新增一个市场时

1. 新建 `<market>/` 目录,按上面的形态搭一条流水线(可从 `china-a/` 拷贝改造)。
2. 配好该目录的 `.env` 与 `service_account.json`,补一个 `.gitignore`。
3. 写该市场的 `README.md`,并在本文件「市场模块」表里加一行。
